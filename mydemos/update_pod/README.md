用法：

该脚本主要用于在k8s集群中更新微服务镜像版本号，并可自动生效deploy配置文件。
1. 使用时，依据当前版本环境的具体参数修改配置信息
2. 配置信息中，base_path为各服务公共目录的最深一层，log_file为日志文件
3. 使用前必须保证各个服务deploy文件的上一层目录名与镜像服务名一致
    如: /data/…/wbs-web/deploy.yml   ==>   wbs-web:1234567890
4. 将该脚本复制到服务器中使用：
    1）在/usr/bin目录下创建文件如：uppod（该文件名可直接作为linux命令使用，可尽量简单，但不要与其他系统命令重复，也不要太易混淆）
    2）将代码复制到uppod文件中保存
    3）执行命令chmod +x uppod，赋予该文件可执行权限；
    4）执行uppod，操作更新
注意：每个项目服务器上创建一个文件就好，不需要每个人上传一个！！！
